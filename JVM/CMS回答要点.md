# CMS
1. 设计目标：最低停顿时间，降低用户程序的停顿时间，加快响应速度
2. 采用的垃圾收集算法：标记清除，导致垃圾碎片，

3. 四个过程
    - 分别是如何查找的情况

**CMS的优势所在：牺牲吞吐量而追求最短停顿时间， 如何做到降低停顿时间的,回答这个问题主要从两个方面来回答**

1. 采用标记-清楚算法，没有进一步采用compact算法，
2. 并且真个垃圾收集是分很多个阶段，mark- swap 分为多个阶段，并且大部分的阶段GC线程都是和用户线程一起并发执行

4. 缺陷
    - 1. cpu敏感型，
    - 2. 浮动垃圾 导致并发模式失败，
    - 3. 垃圾碎片过多，导致没有可用的连续存储空间为大对象，导致full gc
    - 4. gc日志，promotion failure 或者concurrent mode failure 导致gc
