# 协程

与进程和线程最大区别就是：
1. 协程是用户程序自己进行控制和调度的，而进程和线程是操作系统内核进行调度,这个也导致协程切换的靠小基本可以忽略不计，基本没有内核切换的开销。
2. 可以不加锁访问全局变量，
3. 内存开销方面也是一个因素，每个 goroutine (协程) 默认占用内存远比 Java 、C 的线程少。 goroutine: 2KB 

进程和线程都是抢占式调度，而协程是利用用户程序进行调度的
